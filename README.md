# 🚀 FlowsBiz - מערכת דיוור וניהול קמפיינים בוואטסאפ (PWA)

מערכת מתקדמת, מעוצבת ורספונסיבית לניהול קמפיינים ודיוור המוני בוואטסאפ באמצעות ה-**Official Meta WhatsApp Business / Cloud API**. 
המערכת נכתבה מאפס תוך הקפדה יתרה על **ארכיטקטורת משושים (Hexagonal Architecture / Ports & Adapters)**, התאמה מלאה למצב ריבוי דיירים (**Multi-Tenant Ready**), ועמידה בסטנדרטים המחמירים ביותר של אבטחה, ציות לכללי ספאם (Opt-in/Opt-out), וממשק משתמש קצה מרהיב בגוון בהיר (Light Glassmorphism PWA).

---

## 🏗️ ארכיטקטורת המערכת (Hexagonal Architecture)

המערכת בנויה על פי עקרונות ה-Clean Architecture והפרדת רשויות מוחלטת בין לוגיקת הליבה העסקית לתשתיות החיצוניות:

```
                  ┌──────────────────────────────────────────────┐
                  │                 Presentation                 │
                  │        (FastAPI REST APIs / PWA Web)        │
                  └──────────────────────┬───────────────────────┘
                                         ▼
                  ┌──────────────────────────────────────────────┐
                  │                    Ports                     │
                  │       (Abstract Repository, Queue, WA)       │
                  └──────────────────────┬───────────────────────┘
                                         ▼
                  ┌──────────────────────────────────────────────┐
                  │                  Core Domain                 │
                  │     (Entities, Unsubscribe, Segmenter)       │
                  └──────────────────────▲───────────────────────┘
                                         │
                  ┌──────────────────────┴───────────────────────┘
                  │                  Adapters                    │
                  │   (SQLAlchemy, SQLite/PG, Celery, Meta API)   │
                  └──────────────────────────────────────────────┘
```

1. **Core Domain (ליבת המערכת):** ישויות טהורות (`app/core/models`) ולוגיקה עסקית מבודדת (מנוע פילוח קמפיינים, מנהל הסרות).
2. **Ports (ממשקים):** הגדרת חוזים מופשטים (`app/ports`) עבור שמירת נתונים, שליחת הודעות ותורים.
3. **Adapters (מתאמים):** מימושים קונקרטיים של תשתיות חיצוניות (`app/adapters`):
   - **Database Adapter:** שימוש ב-SQLAlchemy (Asyncio) עם תמיכה כפולה ב-SQLite (`aiosqlite` לפיתוח מקומי קל) וב-PostgreSQL (לייצור).
   - **Meta WhatsApp Adapter:** אינטגרציה מלאה מול ה-Cloud API של מטא עם מנגנון הדמיה מובנה (Simulation Mock) המאפשר לבדוק את המערכת ללא מפתחות רשמיים.
   - **Queue/Celery Adapter:** מנגנון שליחה מנוהל ברקע עם ויסות קצב מובנה (Rate-Limiting / Throttling של 2 שניות בין הודעה להודעה) לשמירה על דירוג האיכות של מטא ומניעת חסימות ספאם.

## ✨ תכונות מרכזיות במערכת

- **👥 ייבוא אנשי קשר חכם מ-CSV:** העלאת קבצים מהירה עם מיפוי עמודות חכם (תומך בכותרות בעברית ובאנגלית כמו `טלפון`/`שם`), ניקוי ותיקון מספרי טלפון לפורמט בינלאומי `E.164` (הוספת קידומת `+972`), ושמירה דינמית של עמודות נוספות בשדה `custom_attributes` בפורמט JSON.
- **📊 לוח בקרה מנהלים (Campaign Dashboard):** ניתוח נתונים בזמן אמת, אחוזי המרה, יחס הקלקות, סטטוס שליחות (נשלח, נקרא, נכשל), וחלוקת אנשי קשר לפי סגמנטים.
- **📱 סימולטור צד משתמש קצה אינטראקטיבי (User Simulator Window):** הדמיה מלאה של מכשיר סמארטפון המציג את ההודעות המתקבלות בזמן אמת. מאפשר למשתמש לענות "הסר" או "Unsubscribe", מה שמפעיל אוטומטית את מנגנון ה-Webhook של המערכת, מסיר את המשתמש מרשימת התפוצה ומעדכן את המדדים בלוח הבקרה בזמן אמת.
- **⚙️ הגדרות פרופיל לקוח/דייר (Multi-Tenant Management):** ניהול הגדרות ה-Meta Cloud API (מספר טלפון, מזהה חשבון עסקי WABA, מפתח גישה קבוע) ברמת הדייר.
- **🔒 אבטחה וציות לספאם:** סינון אוטומטי של הודעות לנמענים שהסירו את עצמם (`opt_in_status = "withdrawn"`) ותיעוד מלא של פעולות במערכת ב-Audit Trail.

## 🛠️ טכנולוגיות בשימוש

### Backend (צד שרת)
- **FastAPI** (Python 3.11+) לפיתוח API אסינכרוני מהיר במיוחד.
- **SQLAlchemy 2.0 (Async)** עם מנוע **aiosqlite** לעבודה מול מסד נתונים מקומי ללא התקנות.
- **Uvicorn** לניהול שרת ה-API.
- **Celery / Redis** לניהול תורי משימות ושליחה מתוזמנת/מבוזרת ברקע (כולל מנגנון הדמיית שליחה אסינכרונית ללא תלות ישירה בשרת Redis ייעודי בעת פיתוח).

### Frontend (צד לקוח)
- **React + Vite** עם **TypeScript** לביצועים מהירים וחוויית פיתוח מודרנית.
- **Vanilla CSS (Premium Light Glassmorphic Theme):** עיצוב יוקרתי, רך ובהיר עם אפקטים מרהיבים של זכוכית, מעברי צבעים חלקים ומיקרו-אנימציות לממשק משתמש קצה ולוח הניהול.
- **PWA (Progressive Web App):** תמיכה מלאה בהתקנה כיישום מקומי על גבי טלפונים ניידים ומחשבים.

## 🚀 הוראות הפעלה ופיתוח מקומי

### 1. צד השרת (Backend)

וודא שמותקן אצלך Python 3.11+. מתוך תיקיית `backend`:

```powershell
# 1. ניווט לתיקיית השרת
cd backend

# 2. יצירת סביבה וירטואלית
py -m venv .venv

# 3. הפעלת הסביבה הוירטואלית (ב-Windows PowerShell)
.venv\Scripts\Activate.ps1

# 4. התקנת חבילות ותלויות
py -m pip install -r requirements.txt

# 5. הרצת השרת במצב פיתוח
py -m uvicorn app.main:app --reload
```

- שרת ה-API ירוץ בכתובת: `http://127.0.0.1:8000`
- תיעוד ה-API האינטראקטיבי (Swagger) זמין בכתובת: `http://127.0.0.1:8000/docs`
- מסד הנתונים של SQLite (`whatsapp_db.sqlite`) ייווצר אוטומטית בתוך תיקיית `backend` בהרצה הראשונה ויריץ מיגרציה מלאה אוטומטית.

---

### 2. צד הלקוח (Frontend)

וודא שמותקן אצלך Node.js (גרסה 18 ומעלה). מתוך תיקיית `frontend`:

```powershell
# 1. ניווט לתיקיית הלקוח
cd frontend

# 2. התקנת התלויות
npm install

# 3. הרצת שרת הפיתוח של Vite
npm run dev
```

- הממשק יהיה זמין בדפדפן בכתובת: `http://localhost:5173`
- כל הבקשות ל-`/api/*` מנותבות אוטומטית לשרת ה-FastAPI דרך מנגנון ה-Proxy של Vite המוגדר ב-`vite.config.ts`.

## 🧪 בדיקת המערכת באמצעות ה-Simulators

כדי לאפשר פיתוח מהיר ללא צורך בהתקנת שרתי Redis ותשלום למטא, המערכת כוללת הדמיות מלאות:
1. **שליחת הודעות מדומיינת (Celery/Meta Mock):** בהפעלת קמפיין, המערכת תיכנס למצב שיגור ותציג שורת התקדמות חיה.
2. **סימולטור צד לקוח:** לחצו על תצוגת המשתמש הנייטיבית (בצד שמאל של המסך או תחת עמוד "תצוגת לקוח קצה"). תוכלו לשלוח הודעות "הסר" אל המערכת ולראות כיצד השרת קולט את ה-Webhook, מסיר את הנמען ומעדכן את המדדים בזמן אמת בלוח המחוונים!
